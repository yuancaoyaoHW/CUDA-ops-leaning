# STAR 素材周记

每周周检（Day 7/14/21/28/35/43/49/56）花 30 分钟填一段。W8 用来组装项目深挖问答和 mock，不需要回忆。

模板：

```
## Week N (YYYY-MM-DD ~ YYYY-MM-DD)

### Situation
本周主线是什么，处于 56 天计划的哪个阶段。

### Task
具体要解决的问题或要交付的产物。一句话能说清楚。

### Action
怎么分析的，做了哪些尝试，最后选了哪条路径，关键决策点是什么。
列 2-3 条最有信息量的细节，避免流水账。

### Result
- baseline: 起点指标（PyTorch / 未优化 / 上一版）
- target: 目标指标
- final: 实际达到的指标
- improvement: 提升比例 / 加速比 / GB/s / TFLOPS
- 学到了什么（1-2 句）

### Badcase
- 哪些 shape / dtype / workload / 场景没有优化好
- 失败实验：试了什么不 work，为什么
- 上线/落地约束：如果要在生产用还差什么
- 下一步怎么验证或修复

### Interview-ready
- 一句话项目介绍
- 一个最适合面试讲的技术点
- 预期被追问的 3 个方向，初步答案要点
```
