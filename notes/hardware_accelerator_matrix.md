# Hardware Accelerator Matrix for 2026 LLM Training and Inference

Last researched: 2026-05-31.

This note compares mainstream AI accelerators and systems that matter for 2026 LLM training, fine-tuning, and inference planning. It is a sourced working matrix, not a benchmark ranking. Treat every field as "vendor-published unless the confidence column says otherwise."

## Confidence Rules

- `official`: vendor product page, data sheet, product brief, user guide, or support document publishes the value directly.
- `semi-official`: vendor keynote, press release, blog, roadmap, or source that is official but not a stable product data sheet.
- `third-party`: non-vendor reporting or reseller claims; these are not authoritative.
- `unknown`: no reliable public source found, or public sources conflict.

For Huawei Ascend 910B and 910C, chip-level public documentation is incomplete. This matrix therefore keeps many chip fields as `unknown` and records only the system-level Atlas specifications that Huawei publishes.

## Scope Notes

- NVIDIA H20 is omitted. It is China-market and export-control sensitive; no official NVIDIA product data sheet with complete specs was used in this pass.
- AMD MI350P is omitted. A credible third-party article reports a PCIe MI350P, but an official AMD product page or data sheet was not available in the sources captured here.
- AMD MI300A is omitted because the task made it optional and the 2026 LLM training/inference comparison is better served by MI300X/MI325X/MI350-series accelerators.
- Several NVIDIA Blackwell and Blackwell Ultra pages publish system-level data more clearly than standalone GPU tables. Where a per-GPU peak table was not found, the matrix says `unknown` instead of deriving unofficial precision rows.
- Intel Gaudi 3 has official pages and a white paper landing page, but the full precision table was not exposed in the text captured here. Memory, bandwidth, form factor, Ethernet/RoCE positioning, and PCIe-card power are sourced from Intel pages/search-result text; compute peaks remain `unknown` unless verified directly from an Intel PDF/table.

- Ascend 910C chip-level memory capacity, memory type, memory bandwidth, FP32, BF16, FP16, FP8, FP4, INT8, INT4, sparsity behavior, and single-chip power.

## Source Coverage Summary

- NVIDIA: strong official coverage for H100, H200, DGX B200, GB200 NVL72, GB300 NVL72, L40S, and L4. Standalone B200/B300 per-precision data is weaker than system-level DGX/HGX/NVL data.
- AMD: strong official coverage for MI300X, MI325X, and MI355X. MI350X has official memory/bandwidth/generation coverage, but full per-precision values were not copied without a directly captured official table.
- Intel: official coverage confirms Gaudi 3 product positioning, 128 GB HBM2e, 3.7 TB/s bandwidth, PCIe card availability, Ethernet/RoCE design, and software integration. Full precision peaks need direct verification from an Intel PDF/table before being treated as official.
- Huawei: official coverage is strongest at Atlas system level, especially Atlas 800I A2, Atlas 800T A2, Atlas 900 A3, Atlas 300I Pro, and roadmap SuperPoDs. Public chip-level 910B/910C coverage is incomplete, so those rows intentionally remain mostly `unknown`.
